웹서버

웹서버는 웹 클라이언트로부터 전송되어 오는 서비스 요청에 대한 결과인 HTML 문서를 해당 웹 클라이언트에 ㅈ공
웹 브라우저 : HTML로 작성된 하이퍼 텍스트 문서를 해석하여, 화면에 출력
가장 대표적인 웹서버로 아파치 사용
-안정성, 확장성을 고려한, Open Source Project

(웹서버 : 파일을 요청하면, 파일을 다운로드 해주는 서버라는 컨셉으로 이해하면 쉽다, 나머지 기능은 브라우저가 한다.)

Python에서는 경량 web server를 위해 세가지 방법을 제공



순서
1. 폴더 하나 생성
2.폴더 내에, html 파일 생성
3.콘솔에 python -m http.server 80


서버에 접속 : 127.0.0.1:80/폴더 내 파일
내 컴퓨터는 지구, 브라우저는 달나라

웹 : 다운로드 받아서, 특정 폴더에 저장만 하는것, 웹서버는 실행하는서버가 없다.

Python에서는 경량web 서버를 위해 3가지 방법 제공

1.기본 웹서버
python -m http.server 9000

2.스크립트 기반 웹서버

3.web Framework 기반 웁서버



웹 서버 어플리케이션
웹을 기반으로 서버 실행되는 프로그램

동적 HTML 생성 : 웹 서버 애플리케이션은 HTML을 리턴
웹 서버 어플리케이션은 파일, 데이터 베이스 등 기존 서버 자원 활동

웹 어플리케이션이 리턴 하면, 웹서버가 브라우저에 전달, 브라우저 입장에서는 웹어플리케이션인지 뭔지 알 수 없다.

브라우저 - 웹 서버 - 웹 어플리케이션
웹서버와 웹 어플리케이션은 로컬 통신
브라우저와 웹 서버간에는 네트워크

요새는 1, 2달 교육받고, 바로 웹 어플리케이션을 만들 수 있을정도로 간단해졌다.
